# 数智化场站系统

## 核心定义

> **数智化场站系统**是以**数字孪生 + 数据驱动 + AI 诊断**三位一体为核心能力，对风光储等新能源场站实现**全域感知、实时映射、智能决策、预测性维护**的新一代运营管理平台。

区别于传统 SCADA（偏数据采集与监视控制）和"智慧场站"（偏物联网与自动化），**数智化的核心差异在于"数据→知识→行动"的闭环**——数据不只是被"看见"，而是被 AI 引擎消化后输出可执行的运维决策。

## 最小心智模型

```
                    ┌─────────────┐
                    │   AI 诊断引擎 │
                    │  (传动链/叶片/  │
                    │  塔筒/功率预测) │
                    └──────┬──────┘
                           │ 决策输出
    ┌──────────┐    ┌──────┴──────┐    ┌──────────┐
    │ 物联感知层 │───▶│ 数据底座     │───▶│ 三维可视化 │
    │ SCADA/振动/ │   │ 时序/清洗/   │   │ 数字孪生   │
    │ 视频/气象   │   │ 特征工程    │   │ 全景映射   │
    └──────────┘    └─────────────┘    └──────────┘
```

三个逻辑层，下层为上层供能，**不可跳过**任一。

## 核心能力

| 维度 | 能力 | 说明 |
|------|------|------|
| **感知** | 多源异构数据接入 | SCADA、振动、视频、气象、无人机、安防 6+ 类数据源统一接入 |
| **映射** | 三维数字孪生 | 风机四层拆解、光伏全域模型、储能电池舱级还原、场区地形重建 |
| **诊断** | AI 故障预测 | 传动链/叶片/塔筒故障早期预警，CNN-Transformer 混合架构 |
| **决策** | 预测性维护建议 | 健康评分、剩余寿命概率区间（P10/P50/P90）、检修建议输出 |
| **展示** | 沉浸式交互 | 大屏驾驶舱、PC 门户、远程巡检台、移动 APP 四端覆盖 |

## 边界辨析

| 对比概念 | 同 | 异 |
|---------|-----|-----|
| **SCADA** | 都接入设备实时数据 | SCADA 止于监视与控制；数智化在此基础上增加了 AI 诊断和数字孪生 |
| **智慧场站** | 都以提升运维效率为目标 | "智慧"偏重自动化与 IoT；"数智化"强调 AI 驱动的**数据闭环** |
| **数字孪生** | 都做三维建模 | 数字孪生是数智化的**一个能力层**而非全部；数智化还需 AI 诊断和运维决策 |
| **传统信息化** | 都有数据看板 | 信息化仅做到"看见"；数智化做到"看懂+预判+建议" |

## 在 JBFG 项目中的定位

冀北风光储数智化生产支撑系统的需求范围覆盖了数智化场站系统的核心要素：

- **整体门户** → 统一入口
- **风光储三维可视化** → 数字孪生映射
- **巡检动态展示** → 感知 + 展示
- **构网型透明场站** → 涉网性能的实时可视化（数智化在电网交互侧的延伸）

**缺失的拼图**：客户需求范围未提及 AI 诊断与预测性维护能力（如传动链诊断、叶片诊断等），这些在[[AI-技术方案-陆上风电场智慧运维与运行监视平台建设方案V1.docx|陆上风电场方案]]中是核心模块。方案书需要对此做出选择：是覆盖完整数智化能力，还是仅做可视化层？
