# 课题分层选择指南

## 三层级

| 层级 | 子目录 | 适用场景 |
|------|--------|---------|
| **轻量** | MOC.md + 过程/ + z附件/ | 探索性调研、临时任务、单次决策类 |
| **标准** | MOC.md + 调研/ + 过程/ + z附件/ | 技术选型、方案设计、领域学习 |
| **重型** | MOC.md + 调研/ + 设计/ + 过程/ + 资产/ + z附件/ | 系统建设、产品设计、多阶段项目 |

## 触发条件

| 条件 | 层级 |
|------|------|
| 预期笔记 ≤ 3 篇 | 轻量 |
| 预期笔记 4-10 篇 | 标准 |
| 预期笔记 > 10 篇或多阶段 | 重型 |
| 用户明确说"轻量/快速/临时" | 轻量 |
| 用户明确说"正式/完整/文档化" | 重型 |

默认选「标准」——调研和过程日志是最小可用的起点。

## 创建闭环

新建课题不是只创建目录和课题 MOC，必须完成上级 MOC 归口。

| 步骤 | 动作 | 验收 |
| --- | --- | --- |
| 1 | 判断承载目录 | 明确进入 `03-课题/` 或其主题子目录 |
| 2 | 判断上级 MOC | 默认 `03-课题/03-课题.md`，有主题 MOC 时挂主题 MOC |
| 3 | 创建课题目录与同名 MOC | 目录、MOC、过程/、z附件/ 已存在 |
| 4 | 回写上级 MOC | 按状态加入「活跃课题 / 待启动 / 资源类 / 外部项目组」 |
| 5 | 验证归口 | `rg` 能在上级 MOC 中检出新课题链接 |

缺少第 4-5 步时，课题创建不算完成。

## 各目录说明

| 目录 | 角色 | 何时有 |
|------|------|--------|
| MOC.md | 课题总览：目标、范围、状态、产出物索引 | 必需 |
| 调研/ | 竞品分析、文献综述、技术可行性笔记 | 标准+ |
| 设计/ | 架构图、接口设计、方案对比评估 | 重型 |
| 过程/ | 决策日志、每日进展、问题记录 | 必需 |
| 资产/ | 代码、配置、图表等可交付产出 | 重型 |
| z附件/ | 截图、外部 PDF、临时文件 | 必需 |

## MOC 分组策略（按课题类型）

| 课题类型 | 推荐分组 | 示例 |
|---------|---------|------|
| 技术调研 | workflow | `## 背景 → ## 方案对比 → ## 结论` |
| 系统建设 | timeline | `## Phase 1 → ## Phase 2 → ## Phase 3` |
| 产品设计 | concept_hierarchy | `## 用户需求 → ## 功能设计 → ## 技术约束` |
| 领域学习 | concept_hierarchy | `## 基础概念 → ## 核心算法 → ## 前沿扩展` |
