# 辨析-GIS服务引擎选型(GeoServer vs SuperMap vs MapGIS)

## 结论

**一期推荐 GeoServer + PostGIS**，客户要求国产化时切换超图 iServer。GeoServer 在 OGC 标准兼容、部署复杂度、社区生态和成本上优于国产商业 GIS，但缺乏企业级门户（Portal）和开箱即用的空间分析服务。超图 iServer 在三维场景服务（S3M）、国产化信创适配、中文技术支持和开箱即用能力上优于 GeoServer，但授权成本高且架构较重。中地 MapGIS 在地质矿产领域有深厚积累，但开放性和推广度不如超图。

---

## 一、核心维度对比

### 1.1 功能覆盖

| 功能域 | GeoServer | 超图 iServer | 中地 MapGIS |
|--------|-----------|-------------|------------|
| OGC 标准服务（WMS/WFS/WCS/WMTS） | ✅ 原生支持，最完整 | ✅ 完整支持 | ✅ 支持 |
| REST 风格 API | ⚠️ 社区扩展，非原生 | ✅ 原生 REST 数据/地图/三维服务 | ✅ 原生支持 |
| 三维场景服务 | ❌ 不支持，需外接 Cesium | ✅ S3M 标准，三维场景服务原生 | ✅ 三维分析能力 |
| 空间分析服务 | ⚠️ 仅 WPS 基础分析 | ✅ 内置缓冲区/叠加/插值/网络分析 | ✅ 地质专业分析 |
| 门户管理（Portal） | ❌ 无原生 Portal，仅管理后台 | ✅ iPortal——权限/共享/低代码 | ✅ 有门户组件 |
| 服务聚合（多源异构） | ❌ 不支持 | ✅ 可聚合 GeoServer 服务 | ⚠️ 有限 |
| 瓦片缓存 | ✅ GeoWebCache 内置 | ✅ 多格式瓦片（SMTiles/UGC） | ✅ 支持 |
| 数据格式支持 | PostGIS/Shapefile/GeoTIFF | 100+ 格式 + UDBX 引擎 | 专业地质数据格式 |

### 1.2 性能

| 场景 | GeoServer | 超图 iServer | 中地 MapGIS |
|------|-----------|-------------|------------|
| 矢量瓦片发布（百万级要素） | ✅ GeoWebCache 预切片，静态瓦片性能高 | ✅ 多级缓存 + 分布式集群，大规模并发更优 | ✅ 海量数据管理有优势 |
| 动态 WMS 渲染（实时样式） | ⚠️ 单请求 50-200ms，高并发需集群 | ✅ 集群化部署 + 多级缓存，并发性能更强 | ⚠️ 取决于部署配置 |
| WFS 要素查询（复杂过滤） | ✅ PostGIS 索引 + SQL 下推，性能好 | ✅ REST 数据服务 JSON 格式，交互简洁 | ✅ 数据库优化好 |
| 三维场景发布 | ❌ 不在能力范围 | ✅ S3M 瓦片流式加载 | ✅ 三维分析优化 |

**性能关键认知**：GeoServer 的性能瓶颈不在自身渲染引擎，而在后端数据源和缓存策略。正确配置 GeoWebCache + PostGIS 空间索引后，静态瓦片服务性能三者差距不大。超图的优势在于集群化分布式部署的开箱即用能力。

### 1.3 稳定性

| 维度 | GeoServer | 超图 iServer | 中地 MapGIS |
|------|-----------|-------------|------------|
| 社区验证 | ✅ 全球数万生产部署，20 年历史 | ✅ 国内外中大型项目验证 | ⚠️ 主要集中在地质/测绘领域 |
| 版本迭代 | 稳定版 6 月/次，长期维护版 LTS | 年度大版本 + 季度补丁 | 年度版本 |
| 故障恢复 | 依赖外部方案（Nginx HA / K8s） | 内置集群管理 + 故障转移 | 内置集群管理 |
| 内存管理 | JVM GC，需调优 | 自研 JVM 优化 | 自研引擎 |
| 安全漏洞响应 | 社区快速响应，CVE 公开 | 商业支持，不公开 CVE | 商业支持 |

### 1.4 国产化与信创

| 维度 | GeoServer | 超图 iServer | 中地 MapGIS |
|------|-----------|-------------|------------|
| 国产化认证 | ❌ 非国产（美国 OSGeo 基金会） | ✅ 完全国产，信创名录 | ✅ 国产，信创适配 |
| 国产 OS 兼容 | ✅ 可运行（JVM 跨平台） | ✅ 麒麟/统信官方认证 | ✅ 麒麟/统信官方认证 |
| 国产数据库适配 | PostGIS（非国产） | ✅ 达梦/人大金仓/瀚高 | ✅ 达梦/人大金仓 |
| 采购合规 | 无限制（开源免费） | 需商业授权采购 | 需商业授权采购 |
| 信创验收 | ⚠️ 可能被要求替换 | ✅ 通过信创验收 | ✅ 通过信创验收 |

**关键判断**：若客户最终明确要求 GIS 服务国产化，GeoServer 是合规风险点。但开源 GeoServer 作为一期方案的过渡成本几乎为零——切换到超图 iServer 的适配成本在于 OGC→REST 数据服务接口迁移，约 2-3 人周。

### 1.5 成本

| 成本项 | GeoServer | 超图 iServer | 中地 MapGIS |
|--------|-----------|-------------|------------|
| 许可费用 | **0**（GPL 开源） | 约 15-40 万/套（含基础扩展） | 约 10-35 万/套 |
| 部署成本 | 低（Docker 单容器即用） | 中（需集群规划 + 中间件） | 中 |
| 开发成本 | 中（Portal 需自研或外接） | 低（iPortal 开箱即用） | 中 |
| 运维成本 | 低（标准 JVM 运维） | 中（需超图认证工程师） | 中（需中地认证工程师） |
| 总 TCO（3 年） | **低** | **中高** | **中** |

## 二、选型决策矩阵

| 决策条件 | 推荐方案 |
|---------|---------|
| 无国产化硬性要求 | **GeoServer + PostGIS** |
| 客户明确要求 GIS 国产化 | **超图 iServer + iPortal** |
| 项目为地质/矿产/资源领域 | **中地 MapGIS**（专业优势明显） |
| 需要原生三维场景服务 | **超图 iServer**（S3M 标准） |
| 预算紧张 + 技术团队强 | **GeoServer**（用开发能力换授权成本） |
| 预算充足 + 要求开箱即用 | **超图 iServer** |

## 三、JBFG 推荐策略

**一期：GeoServer + PostGIS（默认）+ 超图备选**

- 一期为单场站，GIS 服务负载低（空间查询 ≤200 QPS），GeoServer 性能绰绰有余
- 提前在架构中抽象 GIS 服务接口（统一 OGC 标准），确保未来切换超图仅改适配器层
- 若客户在评审中提出国产化要求，方案书中已有超图备选方案，可当场切换

**切换到超图的成本估计**：
- OGC WMS/WFS/WMTS → REST 数据服务接口迁移：2-3 人周
- 三维 S3M 服务集成（替代 GeoServer + Cesium 直连）：1-2 人周
- 超图 iPortal 部署 + 权限对接：1 人周
- 合计：约 4-6 人周，一期 4 个月工期内可消化
