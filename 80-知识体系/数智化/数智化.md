# 数智化

## 核心定义

> **数智化**不是"数字化 + 智能化"的简单叠加，而是一种**以数据为原料、以模型为引擎、以决策为产出**的新型运营方法论——将业务问题转化为数据问题，再将数据问题的答案转化为业务行动。

三个否定，划出边界：
- **不是** IT 系统建设（信息化）—— 不关注"流程有没有上线"
- **不是** 数据可视化展示（数字化）—— 不满足于"人看见了"
- **不是** 单纯引入 AI 算法（智能化）—— 不追求"模型精度最高"，追求"决策质量最高"

最终检验标准：**系统产出的决策建议，是否被一线人员信任并执行？**

## 内涵拆解：四层结构

```
         ┌──────────────────────────────────┐
         │       数智化 (Digital-Intelligence)        │
         ├──────────────────────────────────┤
 顶层    │  ① 数据采集与治理（底座）                    │
 术      │  └─ 采、存、通、治、用 五字诀                  │
         ├──────────────────────────────────┤
         │  ② 模型化（核心引擎）                         │
 中层    │  └─ 将业务问题映射为数学问题                    │
 道      │     诊断模型 / 预测模型 / 优化模型 / 决策模型       │
         ├──────────────────────────────────┤
         │  ③ 决策闭环（价值出口）                        │
 底层    │  └─ 模型输出 → 人确认 → 行动 → 反馈 → 模型迭代      │
 法      │     （没有闭环的数智化是"展示型数智化"）              │
         ├──────────────────────────────────┤
         │  ④ 组织适配（必要条件）                        │
 根      │  └─ 信任模型的文化、理解概率的管理、数据驱动的流程       │
         └──────────────────────────────────┘
```

第 ④ 层是最大瓶颈——多数项目在 ①② 层投入充分，到了 ③ 层人不敢信、不肯用。

## 术语选择即战略表态

写"数字化"意味着侧重基础设施和系统建设；写"数智化"意味着强调 AI 驱动和数据闭环。JBFG 方案书策略：封面用"数智化"定调愿景，正文用"数字化"描述本次交付范围，在扩展章节中说明"数智化能力预留"。
