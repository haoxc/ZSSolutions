# ZSSolutions 知识管理 Vault

> 以 **解决方案撰写** 为核心的 Obsidian 项目空间。当前主课题：**JBFG — 数智化生产支撑系统**。

## 项目结构

```
ZSSolutions/
├── readme.md                 ← 本文件，项目入口说明
├── 03-项目课题/               ← 课题全生命周期管理
│   └── 26.0618-数智化生产支撑系统(JBFG)/
│       ├── 03-需求分析/        ← 需求分析产出物
│       └── 复盘/              ← 课题级复盘
├── 80-知识体系/               ← 领域知识笔记（项目无关，可复用）
│   ├── 数智化/               ← 数智化转型、数字平台调研
│   ├── 构网型/               ← 构网型逆变器、GFM 技术
│   ├── 可视化与地理信息/       ← GIS、3D 引擎、可视化
│   ├── 风光储/               ← 风电/光伏/储能一体化
│   └── AI工具/               ← Claude Code skills、工具链
├── 97-复盘/                  ← 跨项目复盘与知识管理改进
└── 99-设置/                  ← 项目配置、脚本、模板
    ├── scripts/
    └── specification/
```

## 核心规范

| 规范 | 说明 |
|------|------|
| **笔记命名** | 中文(English, Abbr).md（术语笔记）／ 辨析-主题(A vs B).md（辨析笔记） |
| **MOC 命名** | 目录名 = 文件夹名（Folder Note 惯例） |
| **课题管理** | 启动三问（目标/产出物/截止日期）+ 烂尾检测（3 天阈值） |
| **知识归属** | 项目无关知识放 `80-知识体系/`，别名不加项目代码 |

详细规则见 `99-设置/specification/` 及 CLAUDE.md 全局指令。
