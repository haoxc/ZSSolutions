# 方案书撰写执行计划

> 目标：将 [[冀北风光储数智化生产支撑与数字孪生解决方案-大纲]] 的 8 章骨架填充为完整方案书正文，通过进度文件实现跨会话自动持续推进。

---

## 一、执行策略：分层闸门

| 层级 | 章节 | 闸门规则 | 理由 |
|------|------|---------|------|
| **L1 核心章** | 第 1 章 项目概述 | 写完自动自检 → 自检通过自动继续下一章 | 素材充足，方向明确 |
| **L1 核心章** | 第 2 章 需求分析 | 同上 | 功能分解清单 + 非功能指标已就绪 |
| **L2 架构章** | 第 3 章 总体技术方案 | 写完**暂停等待确认** | 技术选型影响后续所有章节，方向一错全错 |
| **L1 核心章** | 第 4 章 业务子系统 | 写完自动自检 → 继续 | 5 个子系统相对独立，逐一填充 |
| **L1 核心章** | 第 5 章 非功能特性 | 同上 | 指标数据已备齐 |
| **L1 核心章** | 第 6 章 实施计划 | 同上 | 模板化程度高 |
| **L1 核心章** | 第 7 章 风险管控 | 同上 | 18 项风险已识别 |
| **L3 报价章** | 第 8 章 报价与服务 | 写完**暂停等待确认** | 报价直接影响投标，必须人工审核 |

> **闸门逻辑**：L1 自动滚动，L2/L3 硬暂停。即 8 章中有 2 个确认点（第 3 章和第 8 章），其余自动推进。

## 二、章节依赖与撰写顺序

```
第1章 概述 ──→ 第2章 需求分析 ──→ 第3章 总体技术方案 ──→ 第4章 业务子系统
                                        │                      │
                                        └──→ 第5章 非功能      │
                                        └──→ 第6章 实施计划    │
                                        └──→ 第7章 风险管控    │
                                                              │
                                       第8章 报价与服务 ←──────┘
```

- **第 3 章是关键路径**：技术选型确定后，第 4-7 章可并行撰写
- **第 8 章依赖第 4 章**：报价范围需要明确业务子系统内容

## 三、每章完成标准

| 标准项 | 说明 |
|--------|------|
| **来源标注** | 每节标注数据来源（指向 `01-原始材料/` `02-参考资料/` `03-需求分析/` 中的具体文件） |
| **指标可追溯** | 每个承诺指标可追溯到 [[非功能指标定义]] 或 [[功能分解清单]] |
| **关联链接** | 跨章引用使用 `[[WikiLink]]`，Obsidian 可解析 |
| **字数** | 每章 1500-3000 字（不含图表），报价章 500-800 字 |
| **无占位符** | 无 TODO / TBD / 待填充 标记 |

## 四、进度追踪机制

### 进度文件

`03-项目课题/26.0618-数智化生产支撑系统(JBFG)/20-解决方案/撰写进度.md`

```markdown
## 当前进度
- 状态: writing
- 当前章: 第X章
- 上次更新: YYYY-MM-DD HH:MM

## 章节状态
| 章 | 状态 | 完成时间 | 备注 |
|----|------|---------|------|
| 1 项目概述 | ⬜ 待写 / 🔄 撰写中 / ✅ 已完成 / ⏸ 等待确认 | | |
| 2 需求分析 | ⬜ | | |
| ... | | | |
```

### 恢复流程

每次新会话启动时：
1. 读取 `撰写进度.md` → 识别 `当前章` 和 `状态`
2. 若状态 = `writing` → 读取上一章输出，自检通过后继续下一章
3. 若状态 = `⏸ 等待确认` → 提示用户确认对应章节
4. 若全部完成 → 报告完成

## 五、执行指令模板

用于 `/loop` 或手动触发的指令：

```
读取撰写进度.md，找到当前待写的章节。

若当前章状态为 ⏸ 等待确认：
  - 告知用户当前章节等待确认，暂停

若当前章状态为 ⬜ 待写：
  1. 读取大纲中该章的骨架要求
  2. 读取该章关联的源文件（参考大纲中标注的来源）
  3. 撰写该章完整正文，满足完成标准
  4. 自检：逐项核对完成标准
  5. 更新进度文件，标记 ✅ 已完成
  6. 若该章是 L2/L3 闸门章 → 标记 ⏸ 等待确认，暂停
  7. 否则 → 继续下一章
  8. 若本章是最后一章 → commit 全部输出，报告完成
```
