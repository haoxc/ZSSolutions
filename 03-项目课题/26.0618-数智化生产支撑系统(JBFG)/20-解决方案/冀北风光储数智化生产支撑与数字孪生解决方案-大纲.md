# 冀北风光储数智化生产支撑与数字孪生解决方案

> 方案定位：**USDF 统一空间数据底座——新能源场站的空间对象操作系统**。一台设备注册一次，所有业务按需消费。毫秒级实时驱动，AI 模型即插即用。
> 本文为方案书章节骨架。各章括号内标注数据来源和待写入内容。

## 第一章 项目概述

### 1.1 项目背景
- 冀北风光储基地概况（场站规模、机组配置、并网情况）
- 新型电力系统对场站运维的新要求
- 来源：500009687 技术规范书 + 客户需求范围

### 1.2 方案定位
- **USDF 统一空间数据底座——新能源场站的空间对象操作系统**。一台设备注册一次，所有业务按需消费。毫秒级实时驱动，AI 模型即插即用。
- 四个架构约束：

| 属性 | 架构含义 | 检验标准 |
|------|---------|---------|
| **可靠** | 暂态控制独立通道，安全分区物理隔离 | GOOSE ≤100ms，I 区防误绝不妥协 |
| **扩展** | USDF 天然多场站，一期预留 `station_id` | 二期不加字段、不改 API |
| **AI 友好** | 北向 API 输出标准化对象 + 时序数据 | 算法团队不碰采集层 |
| **实施效能** | 边云协同 + 断网自持 + 国产化 P0 直选 | 一期部署不依赖二期基础设施 |

- **量化价值推导**（从 34 项非功能指标反推，非引用行业通用数字）：

| 价值维度 | 指标推导 | 量化收益 |
|----------|---------|---------|
| 运维人效 | AI 检出率 ≥80% + 告警准确率 ≥95% → 减少无效巡检 | 人工巡检频次降低 60%，运维人效提升 40%+ |
| 故障响应 | 暂态控制 ≤100ms + 端到端 ≤1s → 故障自愈窗口 | 暂态事件人工介入率降低 80% |
| 发电损失 | 光伏故障识别 ≥97% + 储能四级预警 → 减少非计划停机 | 年故障损失电量降低 15–20% |
| 平台扩展 | 低代码组态 + `station_id` 预留 → 新场站即插即用 | 二期场站接入工作量降低 70% |
| 系统可用 | MTBF ≥10,000h + 断网自持 ≥2h → 减少非计划停运 | 年非计划停运 ≤0.88 次 |

- 来源：[[平台建设思想-统一空间数据底座(USDF)]]# 二、方案定位；[[非功能指标定义]]# 六、指标覆盖总览

### 2.1 现状与挑战（Situation + Complication）
- 冀北基地运维现状：多源异构设备、多系统并行、数据孤岛

- 11 个业务模块 × 109 项功能要求（来源：功能分解清单）

## 第三章 总体技术方案

### 3.1 设计理念
- **USDF 底座 + 五业务域**：统一空间数据底座管对象注册/空间参照/属性标准化，五个业务系统管业务逻辑
- 四个架构约束驱动全部技术决策：可靠（三通道分流 + 安全分区）、扩展（`station_id` 预留）、AI 友好（标准 API + 模型即插即用）、实施效能（边云协同 + 国产化）
- 架构原则：底座先行 → 业务渐进 → 多场站可扩展
- **平台级低代码**：F1.1.4 低代码组态是平台扩展性的核心支柱——普通技术人员可配置页面、定义模型、编排工作流，新业务模块上线无需开发团队介入

### 3.2 总体架构

**逻辑四层**：采集层 → 平台层（USDF + 数据引擎）→ 业务层（F1-F5）→ 展示层（三维 Web + 大屏 + 移动端）

**部署拓扑（三级）**：

```
中心云（集团侧）
  GPU 集群（AI 训练）   大数据集群（ClickHouse/Postgres）   全局驾驶舱
        ↕ 广域网（4G/5G/专线）
场站边缘（冀北基地）
  ┌─────────────────────────────────────────────────────┐
  │ 安全 I 区（控制区）        │ 安全 II 区（非控生产区）      │ 安全 III 区（管理信息区）  │
  │                           │                          │                          │
  │ GOOSE 暂态控制 ≤100ms     │ IEC104/Modbus 实时采集     │ 三维 Web 门户             │
  │ 防误保护                    │ Redis 实时缓存             │ 运维工单/报表/大屏          │
  │ 本地兜底                    │ GPU 边缘推理（CV 巡检）     │ AI 训练结果下发             │
  │                           │ AGC/AVC 控制               │                          │
  │                           │ 断网自持 ≥2h               │                          │
  └─────────────────────────────────────────────────────┘
        ↕ 正向隔离装置              ↕ 防火墙
设备层
  风机 SCADA / 光伏监控 / 储能 EMS / SVG & 升压站 / 视频摄像头 / 无人机
```

- **来源**：[[平台建设思想-统一空间数据底座(USDF)]]# 3.4 / # 3.3 / # 4.3

### 3.3 关键技术选型与对比
| 决策域    | 候选方案                                | 推荐  | 理由                       |
| ------ | ----------------------------------- | --- | ------------------------ |
| 三维引擎   | UE5 / Unity / Three.js+Cesium       | 待定  | 需权衡照片级精度 vs 国产化 vs Web 化 |
| 时序数据库  | InfluxDB / TimescaleDB / ClickHouse | 待定  | 百万测点写入基准测试               |
| GIS 引擎 | GeoServer / 超图 / 中地                 | 待定  | Q7 客户需 GIS 服务由我方提供       |
| AI 框架  | 开源 CV + 自训练 / 调用大模型 API             | 待定  | P15 误检率 ≤30% 约束          |
| 流媒体    | RTSP/GB28181 直转 / 转码服务              | 待定  | P24 300 视频终端 + 1000 监测终端 |


### 3.4 安全架构
- 等保二级：安全分区 + 横向隔离 + 纵向认证
- 国产化适配：服务器/OS/数据库/中间件国产化清单

### 3.5 AI 引擎架构
- **三层架构**：边缘推理（场站 II 区 GPU）+ 云端训练（中心云 GPU 集群）+ 人机协同闭环
- **数据就绪**：所有训练/推理数据通过 USDF 北向 API 获取，算法团队不碰采集层
- **模型范围**：一期覆盖 CV 巡检类模型（烟火识别/通道隐患/设备异常），架构为 NLP/大模型预留接入能力
- **关键指标**：单次推理 <500ms / 检出率 ≥80% / 误检率 30%→15% 优化路线图
- 来源：[[平台建设思想-统一空间数据底座(USDF)]]# 4.8

### 3.6 数据治理体系
- **四层治理**：主数据管理（USDF 已有）→ 元数据目录 → 数据血缘 → 数据质量
- **核心规则**：
  - 新增测点自动注册到元数据目录，避免「暗数据」
  - 全链路字段级血缘：采集→Kafka→存储→API，支持回溯
  - 质量 SLA 按通道分级：P1（暂态）> P2（稳态）> P3（历史）
- **与 USDF 互补**：不可越层规则管数据流向的正确性，数据治理管数据自身的可信度
- 来源：[[平台建设思想-统一空间数据底座(USDF)]]# 4.9

## 第四章 业务子系统方案

### 4.1 一体化数智场站平台（F1）
**核心功能（P0-P1 关键项）**：
- 全要素数据建模（F1.1.1）——灵活对象化建模，不写底层代码
- 多类型数据处理（F1.1.2）——风电/光伏/储能/仿真多源接入
- 低代码组态（F1.1.4）——普通技术人员可配页面
- 三维可视化（F1.2.1-5）——风机/光伏/储能/巡检/大屏，照片级精度
- 高频数据接入（F1.3.1）——1kHz/6.4kHz/12.8kHz 录波

**关键指标**：500 并发 / ≤1s 端到端 / 百万测点实时

### 4.2 功率-能量协同优化（F2）
**核心功能**：
- 稳态多源有功/无功协同（F2.1.3-7）——减少弃风弃光、平抑波动
- 暂态快速调频/调压（F2.2.3-6）——GOOSE 协议 ≤100ms
- 安全防误（F2.1.8）——指令异常/状态异常防误保护

**关键指标**：调频偏差 ±1% / 测频精度 ≤0.003Hz / 频率采样 ≤100ms

### 4.3 构网型透明场站（F3）
**核心功能**：
- 机组并网性能评价（F3.1.1-7）——跟网/构网双模式，7 类指标
- 场站并网性能评价（F3.2.1-4）——AGC/AVC/惯量/一次调频/暂态支撑
- 发电性能评价（F3.3.1-3）——风电/光伏/储能发电量统一管理

**关键指标**：数据采集 ≤3s / 单机评价 ≤10s / 场站评价 ≤30s

### 4.4 智能运维系统（F4）
**核心功能**：
- 升压站自动化巡视（F4.2）——无人值守、温度异常识别
- 光伏智能运维（F4.3）——逆变器诊断/组串分析/灰损评估/无人机巡检
- 输电线路智能运维（F4.4）——通道隐患 AI 识别/电子围栏/智能审图
- 储能区安全运维（F4.5）——内短路/热失控四级预警/一致性分析
- 运维任务管理（F4.6）——例行/特殊/专项/自定义运维 + 告警闭环

**关键指标**：AI 检出率 ≥80% / 光伏故障识别 ≥97% / 告警准确率 ≥95%

### 4.5 数字运维管理平台（F5）
**核心功能**：
- 360° 实时监控（F5.1）——全局/单站/设备三级
- 智能 AI 诊断（F5.2）——多维度告警 + 派单闭环
- 智慧运维门户（F5.3）——个人门户/任务中心/两票管理/值班管理
- 生产运营（F5.4）——报表/物资/档案

## 第五章 系统非功能特性

### 5.1 性能指标承诺
- 34 项指标逐项承诺值 + 分层 SLA（来源：非功能指标定义 §1-§6）
- JBFG 量化价值承诺：运维人效 ↑40%+ / 暂态人工介入 ↓80% / 发电损失 ↓15-20% / 二期接入工作量 ↓70%（详见 1.2 方案定位）

### 5.2 可靠性设计
- MTBF ≥10000h 的架构保障：N+1 冗余 / 故障自动切换 / 数据多副本
- 质保期内持续统计 MTBF，期末达标（来源：RB5）

### 5.3 安全性
- 等保二级：身份认证/访问控制/安全审计/数据加密/入侵检测
- 电力二次安防：安全 I/II/III 区隔离 + 正向隔离装置

### 5.4 可扩展性与开发者生态
- **多场站统一调度**：架构支持二期扩展至 N 个场站，`station_id` 天然隔离
- **功能模块可插拔**：新场站接入无需重构平台，新业务模块低代码配置上线
- **开发者 SDK**：北向 API + gRPC（同步）+ Kafka（异步），Python/Matlab/C++ 多语言 SDK
- **开放 API 生态**：`/api/v1/objects/{id}/*` 标准 RESTful 接口，第三方系统可自主集成
- **低代码建模**：F1.1.4 低代码组态——普通技术人员可配置页面/定义模型/编排工作流，不依赖开发团队

## 第七章 风险管控

### 7.1 技术风险（8 项）
- RT1 百万测点写入瓶颈（🔴）→ Kafka ≥32 分区 + ClickHouse
- RT2 500 并发定义模糊（🟡）→ 三层并发模型 + 合同附件
- RT3 暂态通信 100 节点（🟡）→ 网络专项设计
- RT4 AI 误检率 30%（🟡）→ 优化路线图 30%→15%
- RT5 端到端 ≤1s 含渲染（🟡）→ 分层 SLA 承诺
- RT6 仿真模块未评估（🟡）→ 独立子系统，不纳入本期
- RT7 1kHz 精度依赖外部装置（🟢）
- RT8 200 万张图片存储（🟢）

### 7.2 业务风险（5 项）
- RB1 并发/实时性定义模糊致验收争议（🔴）
- RB2 仿真模块遗漏致报价漏项（🔴）
- RB3 AI 指标不达标致验收失败（🟡）
- RB4 采集精度责任归属纠纷（🟡）
- RB5 MTBF 验收周期不可行（🟡）

### 7.3 实施风险（5 项）
- RI1 仿真模块团队能力缺口（🔴）
- RI2 GOOSE 协议集成复杂度（🟡）
- RI3 百万测点接入工期压力（🟡）
- RI4 500 并发全链路压测环境（🟡）
- RI5 Python+Matlab 双语言 SDK（🟢）

> 以上 18 项风险的详细应对策略见 [[非功能指标定义]] §7。
