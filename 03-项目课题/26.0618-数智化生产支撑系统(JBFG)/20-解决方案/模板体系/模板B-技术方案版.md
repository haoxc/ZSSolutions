# 模板 B：技术方案版（Technical Solution Version）

> 适用于技术评审、技术交流、方案汇报等场景。面向技术专家和评审组，聚焦技术说服力，弱化商务色彩。
> **继承关系**：模板 B 是模板 A 的子集——共享相同的字体/颜色/段落样式家族，但内容结构裁剪，封面简化。

---

## 一、与模板 A 的关系

| 维度 | 模板 A（完整应标版） | 模板 B（技术方案版） |
|------|:------------------:|:------------------:|
| 定位 | 正式投标/应标 | 技术评审/交流 |
| 封面 | 公司品牌 + 保密声明 | 简约风格，仅标题+日期 |
| 页眉 | 公司 Logo + 章标题 | 仅章标题，无 Logo |
| 颜色 | CMYK 全色（蓝色系） | 单色印刷友好（灰度优先） |
| 内容 | 8 章 + 报价 + 公司简介 | 5 章技术核心 + 技术附录 |
| 报价章节 | ✅ 含报价与服务 | ❌ 不含 |
| 公司简介 | ✅ 含公司资质 | ❌ 不含 |
| 附录范围 | 完整（术语+标准+接口+清单+资质） | 精简（仅术语+技术附录） |
| 页码 | 罗马+阿拉伯混合 | 统一阿拉伯数字 |
| 推荐打印 | 彩色 | 黑白/灰度均可 |

### 核心差异说明

与模板 A 相比，模板 B **削减**了：

| 删除内容 | 理由 |
|---------|------|
| 修订记录 | 技术交流文档无需版本追溯 |
| 图表索引 | 篇幅较短，目录已够用 |
| 风险管控章节 | 技术评审关注方案可行性而非风险管理 |
| 报价与服务章节 | 技术交流场景不报价 |
| 公司简介 | 技术评审看内容不看公司背景 |
| 附录 D/E（完整清单） | 简化引用，直接指向需求分析文档 |

## 四、排版差异设计

### 4.1 封面

**简约封面布局：**
1. 顶部色块条（1.5cm 高，深蓝底色 #1B3A5C）：空白或「Technical Proposal」字样
2. 空行（~80pt 留白）
3. 方案标题（28pt 黑体，深蓝）：**冀北风光储数智化生产支撑与数字孪生 技术方案**
4. 分隔线（1pt，深蓝）
5. 版本信息（12pt，宋体，灰色）：V1.0 | 2026年XX月
6. 底部色块条（与顶部对称）
7. 底部文字（9pt，灰色）：Confidential

### 4.2 页眉

- 左侧：无 Logo（纯文字方案名称简写）
- 右侧：章标题
- 底部 0.5pt 浅灰分隔线

### 4.3 颜色

- 正文印刷：灰度友好。标题使用 #1B3A5C 深蓝，黑白打印时自动转深灰，可读性不受影响
- 表格表头：浅灰底（C0 M0 Y0 K15），黑白/彩色皆可
- 超链接：下划线代替颜色区分

### 4.4 页码

- 统一阿拉伯数字：1, 2, 3...
- 从封面后第一页开始编号（封面不算页码）
- 格式：`- 1 -`（两侧短线）
- 位置：页脚居中

## 五、样式设置

完全继承模板 A 的样式家族。修改/新增项：

| 样式 | 模板 A | 模板 B |
|------|--------|--------|
| Cover_Title | 28pt，居中对齐 | 28pt，居中对齐，无副标题 |
| Table_Header | 浅蓝底 | 浅灰底（灰度印刷优化） |
| Hyperlink | 蓝色 + 下划线 | 下划线（无颜色依赖） |
| 在样式统一的前提下，可直接套用模板 A 导出的 .dotx |
