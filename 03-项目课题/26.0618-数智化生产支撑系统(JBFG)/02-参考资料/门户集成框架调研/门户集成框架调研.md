# 门户集成框架调研

> **调研目的**：为 JBFG 数智化生产支撑系统门户集成方案选型提供依据。
> **调研范围**：GitHub 上 8 款知名门户集成/微前端框架。
> **调研维度**：核心原理、功能清单、GitHub 指标、性能、隔离性、生态成熟度。

## 四、JBFG 项目选型分析

### 4.1 项目约束

| 约束条件      | 详情                                           |
| --------- | -------------------------------------------- |
| **技术栈**   | UE5 + Spring Boot + Vue 3 + Vite             |
| **集成场景**  | 门户统一挂载：风机可视化/光伏可视化/储能可视化/巡检管理/告警中心/报表中心/系统管理 |
| **对外集成**  | 与南瑞运维平台页面级双向互嵌（iframe/微前端）                   |
| **交付模式**  | 本地私有化部署                                      |
| **团队规模**  | 中等                                           |
| **国产化要求** | 是（Q10 明确）                                    |

### 4.2 推荐方案

|    优先级    | 框架                    | 推荐理由                                                                                        | 风险                                   |
| :-------: | --------------------- | ------------------------------------------------------------------------------------------- | ------------------------------------ |
| 🥇 **首选** | **wujie（腾讯无界）**       | ① Vite 原生支持（技术栈匹配）；② iframe 级隔离，安全性满足电力系统要求；③ 子应用保活，多页签体验好；④ 与南瑞平台 iframe 互嵌天然兼容；⑤ 腾讯团队持续维护 | 富文本编辑器等特殊场景需额外测试                     |
| 🥈 **备选** | **micro-app（京东）**     | ① Vite 支持；② 接入成本低；③ 类 iframe 体验，改造成本小；④ 京东团队持续维护                                            | 多应用保活后路由状态需自行管理                      |
| 🥉 **考察** | **qiankun（蚂蚁）**       | 社区最成熟，文档最全                                                                                  | ⚠️ 近 3 月零 commit 停维风险；Vite 支持弱，需额外插件 |
| ❌ **不推荐** | **Module Federation** | 性能最优但缺沙箱/路由/通信，需大量自建                                                                        | 不适合异构技术栈 + 独立部署场景                    |
| ❌ **不推荐** | **single-spa**        | 需大量自建能力，不适合中等团队直接使用                                                                         | 上手成本极高                               |

### 4.3 推荐架构：wujie + iframe 混合模式

```
┌─────────────────────────────────────────────────┐
│              JBFG 主应用壳 (Vue 3 + Vite)         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐         │
│  │ 导航菜单  │ │ 用户信息  │ │ 主题切换  │         │
│  └──────────┘ └──────────┘ └──────────┘         │
│  ┌──────────────────────────────────────────┐   │
│  │          wujie 子应用容器                  │   │
│  │  ┌─────┐ ┌──────┐ ┌─────┐ ┌─────┐       │   │
│  │  │风机  │ │光伏   │ │储能  │ │巡检  │ ...   │   │
│  │  │可视化│ │可视化 │ │可视化│ │管理  │       │   │
│  │  └─────┘ └──────┘ └─────┘ └─────┘       │   │
│  └──────────────────────────────────────────┘   │
│  ┌──────────────────────────────────────────┐   │
│  │     iframe 外部集成（南瑞运维平台页面）     │   │
│  └──────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

**关键要点**：
- 内部业务子系统 → wujie 组件加载（保活 + 路由同步 + 数据通信）
- 南瑞运维平台页面 → iframe 互嵌（天然隔离，无需 wujie 桥接）
- UE5 三维场景 → 独立 Web Component，通过 wujie 挂载到门户
